# Canvas2D

Juni exposes a small Canvas2D API as WASM imports. In the browser IDE they draw into the **Preview** pane; under Node they are no-ops (text may log).

## Example

```juni
fn main() -> i32:
    canvas_init(640, 360)
    canvas_clear(0.08, 0.09, 0.12, 1.0)
    canvas_fill_rect(40.0, 40.0, 120.0, 80.0, 0.1, 0.7, 0.5, 1.0)
    canvas_fill_circle(320.0, 180.0, 40.0, 0.95, 0.55, 0.2, 1.0)
    canvas_draw_line(40.0, 40.0, 600.0, 300.0, 2.0, 0.2, 0.8, 0.6, 1.0)
    canvas_fill_text("Juni 2D", 40.0, 160.0, 1.0, 1.0, 1.0, 1.0)
    return 0
```

See `examples/canvas2d_hello.juni`, `examples/canvas2d_lines.juni`, and `examples/game_paddle.juni`.

## API

Colors are `f32` in **0..1** (RGBA). Positions and sizes are `f32` pixels unless noted.

| Call | Args | Behavior |
|------|------|----------|
| `canvas_init` | `w: i32`, `h: i32` | Size and show the preview canvas |
| `canvas_clear` | `r, g, b, a: f32` | Fill the canvas |
| `canvas_fill_rect` | `x, y, w, h, r, g, b, a: f32` | Filled axis-aligned rect |
| `canvas_fill_circle` | `x, y, radius, r, g, b, a: f32` | Filled circle |
| `canvas_fill_text` | `s: str`, `x, y, r, g, b, a: f32` | UTF-8 text at `(x, y)` |
| `canvas_draw_line` | `x1, y1, x2, y2, width, r, g, b, a: f32` | Stroked line segment |
| `canvas_stroke_rect` | `x, y, w, h, width, r, g, b, a: f32` | Stroked rect outline |

These are language intrinsics (like `print` / `sqrt`): no import statement required.

## Sprite pattern (no image loader)

Treat a colored rectangle as a simple sprite — position in `state:` and update in `frame`:

```juni
state:
    sprite_x: f32 = 200.0
    sprite_y: f32 = 160.0

fn main() -> i32:
    canvas_init(640, 360)
    return 0

fn frame(dt: f32) -> i32:
    sprite_x = sprite_x + 80.0 * dt
    if sprite_x > 560.0:
        sprite_x = 40.0
    canvas_clear(0.08, 0.09, 0.12, 1.0)
    canvas_fill_rect(sprite_x, sprite_y, 32.0, 32.0, 0.95, 0.4, 0.3, 1.0)
    canvas_stroke_rect(sprite_x - 2.0, sprite_y - 2.0, 36.0, 36.0, 1.0, 1.0, 1.0, 0.4, 1.0)
    return 0
```

Image/sprites asset loading is planned for a future release.
