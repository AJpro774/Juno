# 3D

Juni v3 ships a small **handle-based** 3D API on WebGPU. Handles are `i32` values returned by the host.

## Example

```juni
let cam: i32 = 0
let box: i32 = 0

fn main() -> i32:
    scene3d_init(640, 360)
    cam = camera3d_perspective(60.0, 1.777, 0.1, 100.0)
    box = mesh3d_box(1.0, 1.0, 1.0)
    mesh3d_set_pose(box, 0.0, 0.0, -4.0, 0.4, 0.6, 0.0)
    return 0

fn frame(dt: f32) -> i32:
    scene3d_clear(0.05, 0.06, 0.1, 1.0)
    mesh3d_rotate(box, dt * 0.4, dt * 0.9, 0.0)
    scene3d_draw(box, cam)
    return 0
```

Use **WebGPU** preview mode in the IDE. See `examples/scene3d_cube.juni`.

## API

| Call | Role |
|------|------|
| `scene3d_init(w, h)` | Size WebGPU canvas + depth |
| `camera3d_perspective(fov, aspect, near, far) -> i32` | Camera handle |
| `mesh3d_box(sx, sy, sz) -> i32` | Box mesh handle |
| `mesh3d_set_pose(mesh, tx, ty, tz, rx, ry, rz)` | Position + Euler radians |
| `mesh3d_rotate(mesh, drx, dry, drz)` | Incremental rotate |
| `scene3d_clear(r, g, b, a)` | Clear color + depth |
| `scene3d_draw(mesh, cam)` | Draw one mesh |

Module-level `let` bindings share state between `main` and `frame`.
