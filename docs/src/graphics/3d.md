# 3D

Juni ships a **handle-based** WebGPU 3D API plus ECS attachments so Play mode can draw authored scenes with `world_draw3d`.

## Quick example (imperative handles)

```juni
let cam: i32 = 0
let box: i32 = 0

fn main() -> i32:
    scene3d_init(640, 360)
    cam = camera3d_perspective(60.0, 1.777, 0.1, 100.0)
    box = mesh3d_box(1.0, 1.0, 1.0)
    mesh3d_set_pose(box, 0.0, 0.0, -4.0, 0.4, 0.6, 0.0)
    return 0

fn frame(dt: f32) -> i32:
    scene3d_clear(0.05, 0.06, 0.1, 1.0)
    mesh3d_rotate(box, dt * 0.4, dt * 0.9, 0.0)
    scene3d_draw(box, cam)
    return 0
```

Use **WebGPU** preview mode in the IDE. See `examples/scene3d_cube.juni` and `examples/projects/scene3d_lit`.

## ECS + editor slice

Author entities with **Transform3D**, **Mesh3D** (box or glTF), **Light3D**, and **Camera3D** in the Hierarchy / Inspector. On **Play**, the IDE switches to WebGPU, materializes GPU handles from the `.jscene`, and re-applies that scene after `world_create()`.

```juni
state:
    cam: i32 = 0

fn main() -> i32:
    scene3d_init(640, 360)
    world_create()
    cam = camera3d_perspective(60.0, 1.777, 0.1, 100.0)
    camera3d_orbit(cam, 0.0, 0.0, 0.0, 0.4, 0.35, 6.0)
    let _l = light3d_directional(0.35, -1.0, -0.45, 1.0, 0.95, 0.85)
    return 0

fn frame(dt: f32) -> i32:
    world_step(dt)
    world_draw3d(cam)
    return 0
```

`world_draw3d(cam)` draws every ECS entity with a `mesh3d` attachment (poses synced from `transform3d`).

### `.jscene` Mesh3D fields

| Field | Role |
|-------|------|
| `primitive` | `"box"` (default) or `"gltf"` |
| `size` | Box extents `[sx, sy, sz]` |
| `gltf` | Asset path under `assets/` |
| `color` | Optional `[r, g, b, a]` material tint |

## API

| Call | Role |
|------|------|
| `scene3d_init(w, h)` | Size WebGPU canvas + depth |
| `camera3d_perspective(fov, aspect, near, far) -> i32` | Camera handle |
| `camera3d_look_at` / `camera3d_orbit` | Camera framing |
| `mesh3d_box(sx, sy, sz) -> i32` | Box mesh handle |
| `mesh3d_set_pose` / `mesh3d_rotate` | Pose |
| `material3d_color` / `material3d_texture` / `mesh3d_set_material` | Materials |
| `light3d_directional` / `light3d_point` | Lights |
| `mesh_load_gltf(path) -> i32` | Load glTF JSON (multi-primitive, NORMAL tint) |
| `scene3d_clear` / `scene3d_draw` | Imperative clear + single mesh |
| `scene3d_set_ambient` / `scene3d_set_fog` | Lighting helpers |
| `mesh3d_attach` / `transform3d_set` | ECS attachments |
| `world_draw3d(cam)` | Draw all ECS `mesh3d` entities |

Module-level `let` / `state:` bindings share state between `main` and `frame`.

## Limits (honest)

Full Godot-parity 3D editing, joints, and 3D physics are out of scope. This slice covers authoring + Play with `world_draw3d`.
